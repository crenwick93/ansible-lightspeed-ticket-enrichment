# Setup — EC2 Monitoring Stack

This directory contains the Terraform and Ansible code that provisions and configures the demo infrastructure.

For full setup instructions, see the [Getting Started](../README.md#getting-started) section in the main README.

## Directory Structure

```
setup/
├── terraform/              # EC2 instance, VPC, Elastic IP, SSH key
│   ├── main.tf
│   ├── variables.tf
│   └── outputs.tf
├── playbooks/
│   ├── setup_monitoring_stack.yml   # Installs httpd, Prometheus, AlertManager, Node Exporter, webhook bridge, CMDB CI
│   ├── inventory/
│   │   └── hosts.yml                # Generated by Terraform (gitignored)
│   └── templates/
│       ├── index.html.j2            # httpd hello-world page
│       ├── prometheus.yml.j2        # Prometheus config
│       ├── alert_rules.yml.j2       # Prometheus alert rules
│       ├── alertmanager.yml.j2      # AlertManager config
│       ├── snow_webhook.py.j2       # AlertManager → ServiceNow webhook bridge
│       └── snow_webhook.service.j2  # Systemd unit for the webhook bridge
└── scripts/
    └── setup-apply.sh               # Wrapper: sources .env, runs setup playbook
```

## What Gets Deployed

| Service | Port | Role |
|---------|------|------|
| httpd | 80 | Hello-world app (the monitored workload) |
| Node Exporter | 9100 | Exposes system and systemd metrics |
| Prometheus | 9090 | Scrapes metrics, evaluates alert rules |
| AlertManager | 9093 | Fires alerts to the webhook bridge |
| Webhook bridge | 5001 | Translates alerts into ServiceNow incidents (linked to CMDB CI) |

## What Happens When httpd Stops

1. Node Exporter reports `node_systemd_unit_state{name="httpd.service", state="active"} == 0`
2. Prometheus fires the `ServiceDown_httpd` alert after 15 seconds
3. AlertManager sends a webhook to the local bridge (port 5001)
4. The bridge POSTs a new incident to ServiceNow via the Table API, linked to the CMDB CI
5. From there, the EDA rulebook and AAP workflow take over (see root README)
